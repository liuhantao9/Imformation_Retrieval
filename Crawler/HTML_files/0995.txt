<p>In <a href="/wiki/Cryptography" title="Cryptography">cryptography</a>, <b>XTEA</b> (<b>eXtended TEA</b>) is a <a href="/wiki/Block_cipher" title="Block cipher">block cipher</a> designed to correct weaknesses in <a href="/wiki/Tiny_Encryption_Algorithm" title="Tiny Encryption Algorithm">TEA</a>. The <a href="/wiki/Cipher" title="Cipher">cipher</a>'s designers were <a href="/wiki/David_Wheeler_(computer_scientist)" title="David Wheeler (computer scientist)">David Wheeler</a> and <a href="/wiki/Roger_Needham" title="Roger Needham">Roger Needham</a> of the <a href="/wiki/University_of_Cambridge_Computer_Laboratory" class="mw-redirect" title="University of Cambridge Computer Laboratory">Cambridge Computer Laboratory</a>, and the algorithm was presented in an unpublished technical report in 1997 (Needham and Wheeler, 1997). It is not subject to any <a href="/wiki/Patent" title="Patent">patents</a>.<sup id="cite_ref-wheeler_1-0" class="reference"><a href="#cite_note-wheeler-1">&#91;1&#93;</a></sup>
</p><p>Like TEA, XTEA is a <a href="/wiki/Block_size_(cryptography)" title="Block size (cryptography)">64-bit block</a> <a href="/wiki/Feistel_cipher" title="Feistel cipher">Feistel cipher</a> with a <a href="/wiki/Cryptographic_key_length" class="mw-redirect" title="Cryptographic key length">128-bit key</a> and a suggested 64 rounds. Several differences from TEA are apparent, including a somewhat more complex <a href="/wiki/Key_schedule" title="Key schedule">key-schedule</a> and a rearrangement of the shifts, <a href="/wiki/XOR" class="mw-redirect" title="XOR">XORs</a>, and additions.
</p>

<p>The changes from the reference source code are minor:
</p>
<ul><li>The reference source code used the <code>unsigned long</code> type rather than the <a href="/wiki/64-bit" class="mw-redirect" title="64-bit">64-bit</a> clean <code>uint32_t</code>.</li>
<li>The reference source code did not use <code>const</code> types.</li>
<li>The reference source code omitted redundant parentheses, using C precedence to write the round function as e.g. <code>v1 += (v0&lt;&lt;4 ^ v0&gt;&gt;5) + v0 ^ sum + k[sum&gt;&gt;11 &amp; 3]</code>;</li></ul>
<p>The recommended value for the "num_rounds" parameter is 32, not 64, as each iteration of the loop does two Feistel-cipher rounds. To additionally improve speed, the loop can be unrolled by pre-computing the values of sum+key[].
</p>
<h2><span class="mw-headline" id="Cryptanalysis">Cryptanalysis</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=XTEA&amp;action=edit&amp;section=2" title="Edit section: Cryptanalysis">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>In 2004, Ko et al. presented a <a href="/wiki/Related-key_attack" title="Related-key attack">related-key</a> <a href="/wiki/Differential_attack" class="mw-redirect" title="Differential attack">differential attack</a> on 27 out of 64 rounds of XTEA, requiring 2<sup>20.5</sup> <a href="/wiki/Chosen_plaintext" class="mw-redirect" title="Chosen plaintext">chosen plaintexts</a> and a <a href="/wiki/Time_complexity" title="Time complexity">time complexity</a> of 2<sup>115.15</sup> (Ko et al., 2004).
</p><p>In 2009, Lu presented a related-key rectangle attack on 36 rounds of XTEA, breaking more rounds than any previously published cryptanalytic results for XTEA. The paper presents two attacks, one without and with a weak key assumption, which corresponds to 2<sup>64.98</sup> bytes of data and 2<sup>126.44</sup> operations, and 2<sup>63.83</sup> bytes of data and 2<sup>104.33</sup> operations respectively.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">&#91;2&#93;</a></sup>
</p>
<h2><span class="mw-headline" id="Block_TEA">Block TEA</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=XTEA&amp;action=edit&amp;section=3" title="Edit section: Block TEA">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Presented along with XTEA was a variable-width block cipher termed <b>Block TEA</b>, which uses the XTEA round function, but Block TEA applies it cyclically across an entire message for several iterations. Because it operates on the entire message, Block TEA has the property that it does not need a <a href="/wiki/Block_cipher_modes_of_operation" class="mw-redirect" title="Block cipher modes of operation">mode of operation</a>. An attack on the full Block TEA was described in (Saarinen, 1998), which also details a weakness in Block TEA's successor, <a href="/wiki/XXTEA" title="XXTEA">XXTEA</a>.
</p>
