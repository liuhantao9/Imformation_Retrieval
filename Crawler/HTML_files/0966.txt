<p>In <a href="/wiki/Cryptography" title="Cryptography">cryptography</a>, the <b>Tiny Encryption Algorithm</b> (<b>TEA</b>) is a <a href="/wiki/Block_cipher" title="Block cipher">block cipher</a> notable for its simplicity of description and <a href="/wiki/Implementation" title="Implementation">implementation</a>, typically a few lines of code. It was designed by <a href="/wiki/David_Wheeler_(computer_scientist)" title="David Wheeler (computer scientist)">David Wheeler</a> and <a href="/wiki/Roger_Needham" title="Roger Needham">Roger Needham</a> of the <a href="/wiki/Cambridge_University_Computer_Laboratory" class="mw-redirect" title="Cambridge University Computer Laboratory">Cambridge Computer Laboratory</a>; it was first  presented at the <a href="/wiki/Fast_Software_Encryption" title="Fast Software Encryption">Fast Software Encryption</a> workshop in <a href="/wiki/Leuven" title="Leuven">Leuven</a> in 1994, and first published in the proceedings of that workshop.<sup id="cite_ref-teapaper_4-0" class="reference"><a href="#cite_note-teapaper-4">&#91;4&#93;</a></sup>
</p><p>The cipher is not subject to any <a href="/wiki/Patent" title="Patent">patents</a>.
</p>

<h2><span class="mw-headline" id="Properties">Properties</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Tiny_Encryption_Algorithm&amp;action=edit&amp;section=1" title="Edit section: Properties">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>TEA operates on two 32-bit <a href="/wiki/Unsigned_integers" class="mw-redirect" title="Unsigned integers">unsigned integers</a> (could be derived from a 64-bit data <a href="/wiki/Block_size_(cryptography)" title="Block size (cryptography)">block</a>) and uses a 128-bit  <a href="/wiki/Key_(cryptography)" title="Key (cryptography)">key</a>. It has a <a href="/wiki/Feistel_network" class="mw-redirect" title="Feistel network">Feistel structure</a> with a suggested 64 rounds, typically implemented in pairs termed <i>cycles</i>. It has an extremely simple <a href="/wiki/Key_schedule" title="Key schedule">key schedule</a>, mixing all of the key material in exactly the same way for each cycle. Different multiples of a <a href="/wiki/Magic_number_(programming)" title="Magic number (programming)">magic constant</a> are used to prevent simple attacks based on the <a href="/wiki/Symmetry" title="Symmetry">symmetry</a> of the rounds. The magic constant, 2654435769 or 0x9E3779B9 is chosen to be ⌊2<sup>32</sup>/<i>ϕ</i>⌋, where <i>ϕ</i> is the <a href="/wiki/Golden_ratio" title="Golden ratio">golden ratio</a> (as a <a href="/wiki/Nothing-up-my-sleeve_number" title="Nothing-up-my-sleeve number">Nothing-up-my-sleeve number</a>).<sup id="cite_ref-teapaper_4-1" class="reference"><a href="#cite_note-teapaper-4">&#91;4&#93;</a></sup>
</p><p>TEA has a few weaknesses. Most notably, it suffers from equivalent keys—each key is equivalent to three others, which means that the effective key size is only 126 <a href="/wiki/Bit" title="Bit">bits</a>.<sup id="cite_ref-kelsey1996_5-0" class="reference"><a href="#cite_note-kelsey1996-5">&#91;5&#93;</a></sup> As a result, TEA is especially bad as a <a href="/wiki/Cryptographic_hash_function" title="Cryptographic hash function">cryptographic hash function</a>. This weakness led to a method for <a href="/wiki/Hacker_(hobbyist)" class="mw-redirect" title="Hacker (hobbyist)">hacking</a> <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a>'s <a href="/wiki/Xbox_(console)" title="Xbox (console)">Xbox</a> <a href="/wiki/Game_console" class="mw-redirect" title="Game console">game console</a>, where the cipher was used as a hash function.<sup id="cite_ref-6" class="reference"><a href="#cite_note-6">&#91;6&#93;</a></sup> TEA is also susceptible to a <a href="/wiki/Related-key_attack" title="Related-key attack">related-key attack</a> which requires 2<sup>23</sup> <a href="/wiki/Chosen_plaintext" class="mw-redirect" title="Chosen plaintext">chosen plaintexts</a> under a related-key pair, with 2<sup>32</sup> time complexity.<sup id="cite_ref-kelsey1997_2-1" class="reference"><a href="#cite_note-kelsey1997-2">&#91;2&#93;</a></sup>  Because of these weaknesses, the <a href="/wiki/XTEA" title="XTEA">XTEA</a> cipher was designed.
</p>
<h2><span class="mw-headline" id="Versions">Versions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Tiny_Encryption_Algorithm&amp;action=edit&amp;section=2" title="Edit section: Versions">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The first published version of TEA was supplemented by a second version that incorporated extensions to make it more secure. <i>Block TEA</i> (which was specified along with <a href="/wiki/XTEA" title="XTEA">XTEA</a>) operates on arbitrary-size blocks in place of the 64-bit blocks of the original.
</p><p>A third version (<a href="/wiki/XXTEA" title="XXTEA">XXTEA</a>), published in 1998, described further improvements for enhancing the security of the Block TEA algorithm.
</p>

        <span class="n">v0</span> <span class="o">-=</span> <span class="p">((</span><span class="n">v1</span><span class="o">&lt;&lt;</span><span class="mi">4</span><span class="p">)</span> <span class="o">+</span> <span class="n">k0</span><span class="p">)</span> <span class="o">^</span> <span class="p">(</span><span class="n">v1</span> <span class="o">+</span> <span class="n">sum</span><span class="p">)</span> <span class="o">^</span> <span class="p">((</span><span class="n">v1</span><span class="o">&gt;&gt;</span><span class="mi">5</span><span class="p">)</span> <span class="o">+</span> <span class="n">k1</span><span class="p">);</span>
        <span class="n">sum</span> <span class="o">-=</span> <span class="n">delta</span><span class="p">;</span>
    <span class="p">}</span>                                              <span class="cm">/* end cycle */</span>
    <span class="n">v</span><span class="p">[</span><span class="mi">0</span><span class="p">]</span><span class="o">=</span><span class="n">v0</span><span class="p">;</span> <span class="n">v</span><span class="p">[</span><span class="mi">1</span><span class="p">]</span><span class="o">=</span><span class="n">v1</span><span class="p">;</span>
<span class="p">}</span>
</pre></div>
<p>Note that the reference implementation acts on multi-byte numeric values. The original paper does not specify how to derive the numbers it acts on from binary or other content.

<ul><li><a href="/wiki/RC4" title="RC4">RC4</a> – A <a href="/wiki/Stream_cipher" title="Stream cipher">stream cipher</a> that, just like TEA, is designed to be very simple to implement.</li>
